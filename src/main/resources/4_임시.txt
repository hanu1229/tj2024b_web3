[ HTTP 응답 코드 ]
    01. 정의 : 응답자(서버)가 요청자(클라이언트)의 요청에 대한 결과를 나타내는 코드
    02. 종류
        1XX : 요청을 받고 프로세스를 계속 진행 중
        2XX : 요청을 받고 응답을 이해, 처리 완료
            200 : OK, 요청을 성공하고 응답을 포함
            201 : Create(저장)요청을 성공하고 응답을 포함
            204 : 요청을 성공하고 응답 값을 포함하지 않을 때
        3XX : 요청을 완료하고 추가 조치를 요구
        4XX : (클라이언트 오류)
            400 : 서버가 이해 할 수 있는 요청 (쿼리스트링, body, 매핑등 오류)
            401 : 서버가 요청에 따른 권한 없음
            403 : 서버가 요청에 따른 권한 거부
            404 : 서버가 요청에 따른 리스소(값/html/매핑 등)를 찾을 수 없을 때
            409 : 서버가 요청에 따른 값이 중복 일때 (아이디 중복 등)
        5XX : (서버 오류)
            500 : 서버의 예기치 못한 오류 발생

[ 스프링에서 응답 코드 정송/반환 가능 ]
    - ResponseEntity<T> : Spring에서 HTTP 응답 객체를 제어할 수 있게 해주는 클래스
        - 참고 : < > 제네릭 타입은 기본타입이 불가능하고 참조타입만 가능
    - 형태
        ResponseEntity<반환 타입> 메소드명() {
            return ResponseEntity.status(응답코드);
        }
    - 방법
        - ResponseEntity.status(응답코드).body(반환값);
        - ResponseEntity.status(응갑코드).build();
